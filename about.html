About This Project

The goal of this project was to create an Android application that would display hymns.

In the end, this would involve downloading and parsing 1,885 web pages for the relevant song content, and serving them through a website to a client application, along with storing them in an SQLite database to allow searching. I basically created an API for the site from scratch, implementing a RESTful service for mobile app consumption.

But why would we need all that? The hymns are available for free at hymnal.net. There is also a mobile download already available, an epub file that can be read by the iSilo reader.

Simple: the website was a pain to use on smartphones, and the iSilo option cost an arm and a leg. It's my philosophy that a website with free content should not force users to buy an app to use it more conveniently on a smartphone. (Hymnal.net was not the only one guilty of this. My next target was originally going to be UltimateGuitar.)

I've now realized that a website-scraping app that displays the content of webpages would never fly with copyright rules. But even as I saw that my app would never be published, I noticed how hymnal.net changed its site from the original clunky design to the current, mobile-friendly design. And I now have hope that in due time, the Web will become smartphone friendly, driven as it is by the market. But I also hope that I'll be able to be a part of that transition.

But What About the Website?

I developed this website myself during my first semester at Berkeley, since I'd been looking to get into web development for a while. I fell in love with jQuery, and realized that documentation of my project would solidify my understanding and serve as a reminder for future projects.

Thus, this site is not meant for others, at least in the traditional sense. SEO interests me, but I have no chance of discovery as a resource to others as it now stands, with an obscure domain name provided by free hosting.
